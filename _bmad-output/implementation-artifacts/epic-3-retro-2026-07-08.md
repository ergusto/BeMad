## What went well

- **Adversarial review caught a real defect in every single story** — and several were *self-introduced* by big refactors: 3.2's per-surface reroll bug (a genuine FR-16 violation), 3.3's unwired `<h1>`/appTitle + Save/Cancel not re-rolling, 3.4's sort `aria-label` re-introducing a 2.5.3 mismatch. Passing unit/E2E did not catch these; the three-layer review did.
- **The "pure core + thin React binding" pattern held for the whole epic** — catalog data, pure `sortEntries`-style selector + controller — so the rotation engine was fully unit-testable **without** finally adding the deferred React Testing Library harness. Another epic delivered with **no new dependencies**.
- **The Epic-2 retro's `data-testid` decision paid off exactly as intended** — migrating ~50 E2E locators off rotating text let us voice every surface without a brittle test rewrite; error-path specs cleanly switched to "voiced, not raw".
- **The differentiator shipped verified**: rotating, bleeped, clear copy + client-side voiced errors + ALL-CAPS-via-CSS, all passing an automated WCAG 2.2 AA axe audit with keyboard support.

## What was harder / lessons

- **A cross-cutting decision flip-flopped between the Epic-2 retro and Story 3.4's AC.** The retro locked "voiced visible text IS the accessible name" (WCAG 2.5.3-clean); 3.3 implemented it; then 3.4's AC explicitly required *stable* accessible names asserted across a re-roll, forcing a reversal to stable `aria-label`s — which re-opened the 2.5.3 tension. **Lesson:** before locking a cross-cutting decision, read the *later* story's ACs that depend on it (3.4's "stable name" requirement was knowable at retro time).
- **Large-refactor stories leak instance-level gaps.** 3.3 (+452 lines) shipped with two "aggregate-true / instance-false" misses (appTitle, Save/Cancel reroll) — exactly the pattern Epic 2's action item warned about. **Lesson (reinforced):** for many-surface stories, work a per-surface checklist; don't trust "all surfaces done" in aggregate.
- **WCAG 2.5.3 (Label in Name) genuinely conflicts with rotating voice.** Stable SR names + rotating visible text can't both contain each other. Resolved as a **documented product trade-off** (stable aria-labels; the axe `label-content-name-mismatch` rule is allowed for the intended rotating controls). This is a standing constraint of the voice concept.
