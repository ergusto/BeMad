## What Went Well

1. **The architecture spine (AD-1…AD-14) gave crisp, compounding guardrails.** AD-5's single shared Zod schema let Story 1.3 validate identically on server and client with no duplication. Building the Story 1.4 `PATCH` route *general* (partial update) made Story 1.5 (toggle) nearly free — deliberate foresight that paid off.
2. **Real dependencies, not mocks.** Repository/route tests ran against real Postgres 18; E2E drove a real Chromium against the running app. Confidence is genuine, not simulated.
3. **The 3-layer adversarial code review earned its keep.** The project-access layers (Edge Case Hunter, Acceptance Auditor) repeatedly *refuted* Blind Hunter false-positives (the "vitest alias is broken", the "malformed-id → 500") while the layers together caught real defects the happy-path tests missed: a load-vs-create race, a toggle+edit race, and a11y label ambiguities.
4. **Scope fences per story prevented over-building and rework**, and **verification-before-completion** was held throughout (measured coverage, live behaviour, executed E2E).

## What Was Hard (Patterns Across Stories)

1. **Toolchain version churn (recurring).** ESLint 10 broke `eslint-config-next` → pinned to 9; Postgres 18 relocated its data dir → mount at `/var/lib/postgresql`; Zod 4 moved `z.uuid()`/`z.iso.datetime()`; Next 16 made route `params` async. **Lesson: verify APIs against the installed version, never from memory/training.**
2. **React async-UI races (recurring).** Load-vs-create, double-submit, toggle+edit each needed ad-hoc guards (in-flight flags, disable-during-op, single-editor). **This validates the Epic 2 plan** — a single client store + optimistic/rollback layer is the systemic fix.
3. **E2E locator fragility (recurring).** `role="alert"` collided with Next's route announcer; per-row `aria-label`s collided with regex name matchers when task text contained control words ("edit"/"cancel"). **Lesson: role-based + exact names, scoped to the row.**
4. **Shared-DB test coupling.** Two integration files clobbered the shared test DB → fixed with `fileParallelism:false`.
5. **Process gaps.** No commits until the end (six stories in one commit); UI failure-paths are only E2E-covered (no component-unit harness); `project-context.md`'s git-init note was stale (corrected).

## Key Takeaways

1. Adversarial multi-layer review + real-dependency tests catch the bugs that matter — keep it.
2. Pin exact versions AND verify their APIs against what's installed; the "latest" pins carry breaking changes.
3. Ad-hoc React state guards don't scale — Epic 2's single store + optimistic layer is the right consolidation.
4. Commit incrementally and invest in a component-test harness so UI logic isn't E2E-only.
